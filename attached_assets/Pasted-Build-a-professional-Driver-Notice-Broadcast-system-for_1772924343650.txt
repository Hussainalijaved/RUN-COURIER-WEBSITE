Build a professional Driver Notice / Broadcast system for Run Courier admin panel and driver app/website using the existing Supabase backend.

IMPORTANT RULES
- Do not change existing Supabase structure unless necessary
- Keep everything synced between website admin, driver website portal, and mobile app
- Notices must be saved in Supabase, not locally
- Only admin users can create, edit, send, or archive notices
- Only approved active drivers should receive notices
- The system must be fast, simple, and professional

FEATURE TO BUILD
Create a new admin feature called:
Driver Notices

This feature must allow admin to:
1. Create notice templates
2. Send a notice to all active approved drivers at once
3. Send a notice to selected drivers only
4. Mark notice as:
   - information only
   - requires acknowledgement
5. Track which drivers have seen the notice
6. Track which drivers have acknowledged the notice
7. Save full history of all sent notices
8. Show notices inside driver dashboard and mobile app
9. Optionally send email notification when a notice is sent
10. Keep a full audit trail with sent date, sender, and recipients

SUPABASE DATABASE
Create the minimum required tables safely.

1) notice_templates
Fields:
- id
- title
- subject
- message
- category (general, legal, payment, rates, system, compliance, emergency)
- requires_acknowledgement (boolean)
- created_by
- created_at
- updated_at
- is_active

2) driver_notices
Fields:
- id
- template_id (nullable in case admin writes one manually)
- title
- subject
- message
- category
- sent_by
- sent_at
- target_type ('all' or 'selected')
- requires_acknowledgement (boolean)
- status ('draft','sent','archived')

3) driver_notice_recipients
Fields:
- id
- notice_id
- driver_id
- driver_email
- delivery_channel ('dashboard','email','push')
- sent_at
- viewed_at
- acknowledged_at
- status ('sent','viewed','acknowledged')

IMPORTANT
- driver_id must link to the existing drivers collection/table
- only drivers with approved status and active status should receive notices
- keep foreign keys correct
- add indexes for notice_id and driver_id for speed
