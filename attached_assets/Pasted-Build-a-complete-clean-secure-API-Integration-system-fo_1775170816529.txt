  - weight

Return:
- success
- booking reference
- internal status
- tracking URL if available
- message

C. GET /api/track/{bookingReference}
Purpose:
Allow API clients to check current booking/job status.

Return:

- pickup complete yes/no
- delivered yes/no

- tracking link if available

D. GET /api/pricing
Purpose:
Return supported vehicle/pricing structure summary if needed for client integration.
Keep this controlled and only show allowed data.

E. GET /api/health
Purpose:
Simple endpoint for client connection testing.
Return success true and environment status.
