- I want it to behave like a normal mobile notification such as SMS / WhatsApp: banner, sound, vibration, and tap to open the job screen

Project stack:
- Mobile app built with Expo / React Native
- Backend uses Supabase
- Jobs are assigned from admin side and stored in Supabase
- The app currently reacts only when open, which means it is using local/in-app logic instead of proper remote push notification flow

3. Generate and save the Expo push token for each logged-in driver in Supabase

5. When admin assigns a job to a driver, trigger a Supabase Edge Function to send a push notification to that driver’s Expo push token
6. Notification must work when app is:
   - open
   - in background
   - fully closed
   - phone locked
7. Add Android notification channel with high importance, sound, vibration, and visible banner
8. Ensure foreground notifications also show properly by setting the notification handler:
   - shouldShowBanner: true
   - shouldShowList: true
   - shouldPlaySound: true
   - shouldSetBadge: true
9. When the driver taps the notification, open the correct screen directly:
   - Job Details screen
   - with the correct job ID passed in notification data
10. Do not create duplicate notifications for the same job assignment
11. Only send notification for newly assigned jobs, not for every status update unless explicitly configured
12. Add error handling and logs for:
   - permission denied
   - missing Expo push token
   - failed push send
   - invalid token
13. Make sure this works with development build / production build, not Expo Go only
14. Keep all data in Supabase and do not break existing login, job assignment, or tracking logic

Implementation details required:

- Set up permission request flow for iOS and Android
- Set up Android notification channel, for example:
  - name: Job Alerts
